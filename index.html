Real-life applications

This problem has many applications. Starting with the problem itself, that is, to place guards or security cameras to cover a place, such as a museum, a bank or a prison, etc. But where its use is most widespread, is in video games. In particular in the programming of artificial intelligence, 3D rendering, lighting effects. And from this comes its use in robotics, special effects and simulation tools.

Visibility Polygon

For a polygon \(P\), a point \(a\) is visible from a point \(b\), if the line segment \(ab\) is entirely inside \(P\). In other words, let imagine a chain of vertices \(Q = (q_1, q_2,..., q_n)\) in the same plane of \(a\) and \(b\), which can be assimilated to the polygon \(P\). \(a\) and \(b\) are visible to each other if the line segment \(ab\) doesn't intersect any line in the chain \(Q\).

Definition of visibility polygon

The visibility polygon from a point \(a\) can be defined as following : Assuming the point \(a\) is inside the polygon \(P\), the visibility polygon is noted as \(VP(P,a)\), which correspond to the set of points in \(P\) visible by \(a\).(TODO : cite ElGindy1981ALA p186-187).

\(VP(P,a) = \{b|b \in P \space and \space ab \space \bigcap \space P = ab\}\).

Implementation of visibility polygon

To compute the vibility polygon for a specific point \(a\), multiple algorithms with different techniques exist. The most common method is by using the circular sweeping method by sending rays from the point \(a\) to all the verticies \(V_i\) of the polygon \(P\). Then removing the rays which cross any edge of the polygon before reaching the corresponding vertex \(V_i\). And for the rays which can continue in the polygon after reaching the vertex \(V_i\), compute the next intersection with the polygon. At the end all the end point of the rays will form the boundaries of \(V(P,a)\).

The algorithm implementend in the illustration 3 bellow, is based on the algorithm explained by Asano Tetsuo (TODO : cite AsanoTetsuo1985EAFF p557-559). He explained how to compute the visibility polygon for a polygon with and without holes, and the complexity at the worst case is \(O(n \space log \space n)\).

Using irrational coordinates for guards

In this section, we will cover the usage of irrational numbers to describe the coordinates of guards. What if irrational numbers can be or cannot be used ?

Monotone polygon

Let us recall the definition of monotone polygon. A polygon P is monotone, with respect to a straight line L, if every line that is perpendicular to L crosses at most 2 times the edges of P.

Rational & irrational lines

Firstly, a rational point is a point whose coordinates are all rational numbers. An irrational point is a point which is not rational. A rational line is a line which contains two rational points. Again, an irrational line is a line which is not rational.

Irrational coordinates for guards are sometimes needed

In this section, we will show, with an example, that there exists a polygon P having integer coordinates for its vertices that can be guarded with 3 guards having irrational coordinates while it is impossible to guard P with less than 4 guards having rational coordinates.
